# Qu'est-ce que le Version Control ?

## Définition
Le version control est un système qui enregistre les modifications d'un fichier au fil du temps, pour pouvoir revenir à une version précédente.

## Pourquoi en a-t-on besoin ?
- Garder un historique de toutes les modifications
- Revenir en arrière en cas d'erreur
- Collaborer sans écraser le travail des autres
- Savoir qui a modifié quoi et quand
- Travailler sur plusieurs fonctionnalités en parallèle

## Exemple concret
Sans Git : tu modifies un fichier, tu fais une erreur, impossible de revenir en arrière.
Avec Git : tu fais un commit à chaque étape, tu peux revenir à n'importe quel moment.
